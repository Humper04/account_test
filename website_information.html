Website Information and Features

Security Measures

  • Password Hashing: Passwords are securely hashed using bcrypt before being stored or logged, ensuring they are never exposed in plain text.
  • Session Management: Sessions are managed securely with session IDs regenerated upon login to prevent session fixation attacks.
  • SQL Injection Prevention: All SQL queries are conducted using prepared statements to prevent SQL injection vulnerabilities.
  • Rate Limiting: Login attempts are limited to prevent brute force attacks, with users locked out after several failed attempts.

User-Friendly Features

  • Consistent Validation: Email and phone number validation are consistently enforced across both registration and profile modification to maintain data integrity.
  • Intuitive Navigation: Direct links between login, registration, and profile modification pages facilitate user navigation and enhance user experience.
  • Clear Error Messages: User feedback does not disclose specific security settings while providing clear, actionable messages. Enhanced error messaging now clarifies when usernames, emails, and/or phone numbers are already in use.

Error Handling and Logging

  • Log Management: Both successful and unsuccessful login attempts are logged with details such as username/email/phone number, hashed password, and IP address for auditing purposes.
  • Input Validation: Input validation for emails and phone numbers is rigorously applied to prevent incorrect data submissions and enhance security.
